**Название:** Bot Multispa

**Каналы работы:**
1. 🌐 Виджет на сайте (десктоп + мобильная версия)

## 2. Функциональные требования

### FR-01: Мультиязычное приветствие
- Бот приветствует клиента одновременно на LV, RU и EN
- Спрашивает предпочитаемый язык общения
- Дальнейший диалог ведётся только на выбранном языке
- Автоопределение языка, если клиент начинает писать без выбора

### FR-02: Консультации по оборудованию
- Информация о трёх категориях оборудования:
  - Строительные леса (рамные/фасадные + модульные)
  - Опалубка перекрытий (система RINGER)
  - Стеновая/фундаментная опалубка (PIONBOX MINI)
- Технические характеристики из базы знаний
- Рекомендации по выбору оборудования

### FR-03: Квалификация клиента (сбор данных)
- Определение цели: аренда или покупка
- **Для аренды:**
  - Тип строительства
  - Наличие чертежей / проекта
  - Список необходимого оборудования
  - Сроки работ
- **Для покупки:**
  - Тип оборудования
  - Количество

### FR-04: Информация о ценах
- Ориентировочные цены аренды из базы знаний:
  - Модульные леса: 2,20 €/м² + ПВН / месяц
  - Фасадные леса: 1,40 €/м² + ПВН / месяц
  - Стеновая опалубка (h=1,20 м): 6,00 €/м² + ПВН / неделю
  - Опалубка перекрытия: 3,00 €/м² + ПВН / неделю
- Оговорка: цена зависит от сложности, точный расчёт — у специалистов

### FR-05: Приём документации
- Приём файлов от клиента (чертежи, проекты) через Telegram и веб-виджет
- Подтверждение получения
- Пересылка менеджеру для проверки

### FR-06: Доставка и сервис
- Информация о доставке на объект
- Информация о консультациях специалистов на объекте
- Специальные условия для постоянных клиентов

### FR-07: Контактная информация
- Телефоны: +371 270 66 380, +371 292 46 838
- Email: siamultispa@gmail.com
- График: Пн–Пт 9:00–18:00
- Сайт: betonaveidni.lv

### FR-08: Сбор контактов клиента
- Предложение оставить телефон или email
- Передача контактов специалистам для подготовки коммерческого предложения

### FR-09: Перенаправление на специалистов
- При сложных/нестандартных вопросах — предложение связаться по телефону или email
- Логирование нерешённых вопросов

### FR-10: Таймаут неактивности
- Через 20 секунд без ответа — проактивное предложение помощи
- Объяснение, какие данные могут понадобиться

### FR-11: Логика диалога (9 шагов)
| Шаг | Действие |
|-----|----------|
| 1 | Приветствие на 3 языках, выбор языка |
| 2 | Таймаут 20 сек → проактивное предложение |
| 3 | Аренда или покупка? |
| 4 | Аренда → детали проекта |
| 5 | Покупка → тип и количество |
| 6 | Доставка и консультации |
| 7 | Сбор контактных данных |
| 8 | Сложные вопросы → перенаправление |
| 9 | Благодарность и прощание |

## 3. Нефункциональные требования

### NF-01: Доступность
- Бот доступен 24/7 по всем каналам
- Время ответа < 3 сек (текст), < 5 сек (телефон с STT/TTS)

### NF-02: Производительность
- Поддержка до 100 одновременных диалогов
- Работа с базой данных без задержек

### NF-03: Масштабируемость
- Архитектура позволяет добавлять новые каналы (WhatsApp, Viber и др.)
- Лёгкое расширение базы знаний

### NF-04: Безопасность
- Защита персональных данных клиентов (GDPR)
- Безопасное хранение файлов (чертежей)
- RLS (Row Level Security) в Supabase

### NF-05: Логирование
- Логи всех диалогов с указанием канала и языка
- Статистика обращений для аналитики

### NF-06: Ограничения поведения
- Без нецензурной лексики
- Без самостоятельных скидок
- Без обещаний, не указанных в БД
- Ответы только на языке клиента
- Без смены темы разговора

## 4. Технологический стек

| Компонент | Технология | Назначение |
|-----------|-----------|------------|
| **Ядро** | Node.js / JavaScript | Бизнес-логика, обработка диалогов |
| **База данных** | Supabase (PostgreSQL) | Единая база знаний, логи, клиенты |
| **Telegram** | Telegraf / grammY | Telegram Bot API |
| **Веб-виджет** | Express.js + Socket.io | HTTP API + чат в реальном времени |
| **Фронтенд виджета** | HTML/CSS/JS | Встраиваемый виджет для сайта |
| **Телефония** | Voximplant / Twilio | Голосовой канал |
| **STT** | Google Speech / Whisper | Распознавание речи |
| **TTS** | Google TTS / ElevenLabs | Синтез речи |
| **NLP/AI** | OpenAI API / LangChain | Понимание намерений, генерация ответов |
| **Деплой** | Railway / Vercel | Хостинг и развёртывание |

## 5. Модель данных (основные таблицы)

### `equipment` — Оборудование
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Первичный ключ |
| category | TEXT | Категория (scaffolding, wall_formwork, slab_formwork) |
| name_lv | TEXT | Название (LV) |
| name_ru | TEXT | Название (RU) |
| name_en | TEXT | Название (EN) |
| description_lv | TEXT | Описание (LV) |
| description_ru | TEXT | Описание (RU) |
| description_en | TEXT | Описание (EN) |
| system_name | TEXT | Система (PIONBOX MINI, RINGER и т.д.) |
| specs | JSONB | Технические характеристики |

### `prices` — Цены
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Первичный ключ |
| equipment_id | UUID | FK → equipment |
| type | TEXT | rent / sale |
| price_per_unit | DECIMAL | Цена за единицу (€) |
| unit | TEXT | m², шт., комплект |
| period | TEXT | month / week / one-time |
| vat_included | BOOLEAN | Включён ли ПВН |
| notes | TEXT | Примечания |

### `clients` — Клиенты
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Первичный ключ |
| name | TEXT | Имя / компания |
| phone | TEXT | Телефон |
| email | TEXT | Email |
| language | TEXT | Предпочитаемый язык (lv/ru/en) |
| channel | TEXT | Канал первого обращения |
| created_at | TIMESTAMP | Дата регистрации |

### `inquiries` — Заявки
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Первичный ключ |
| client_id | UUID | FK → clients |
| type | TEXT | rent / purchase |
| equipment_category | TEXT | Категория оборудования |
| construction_type | TEXT | Тип строительства |
| area | DECIMAL | Площадь (м²) |
| height | DECIMAL | Высота (м) |
| duration | TEXT | Срок аренды |
| has_drawings | BOOLEAN | Есть ли чертежи |
| file_url | TEXT | Ссылка на файл |
| status | TEXT | new / in_progress / completed |
| notes | TEXT | Примечания |
| created_at | TIMESTAMP | Дата создания |

### `dialog_logs` — Логи диалогов
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Первичный ключ |
| client_id | UUID | FK → clients |
| channel | TEXT | telegram / web / phone |
| language | TEXT | Язык диалога |
| message | TEXT | Сообщение |
| role | TEXT | client / bot |
| intent | TEXT | Распознанное намерение |
| created_at | TIMESTAMP | Время сообщения |

### `faq` — FAQ
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Первичный ключ |
| question_lv | TEXT | Вопрос (LV) |
| question_ru | TEXT | Вопрос (RU) |
| question_en | TEXT | Вопрос (EN) |
| answer_lv | TEXT | Ответ (LV) |
| answer_ru | TEXT | Ответ (RU) |
| answer_en | TEXT | Ответ (EN) |
| category | TEXT | Категория вопроса |
