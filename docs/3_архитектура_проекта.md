## Компоненты системы

### 1. Ядро (Core Engine) — `src/core/`

Центральный компонент. **Не зависит от каналов.** Получает унифицированное сообщение, обрабатывает и возвращает ответ.

| Файл | Назначение |
|------|-----------|
| `engine.js` | Главный маршрутизатор: принимает сообщение → определяет язык → распознаёт intent → запускает сервис → формирует ответ |
| `language.js` | Определение языка клиента, хранение выбора, переключение |
| `intents.js` | Распознавание намерений: price_inquiry, equipment_info, rental, purchase, faq, contact, document_upload |
| `dialog.js` | Конечный автомат (FSM) — управление 9 шагами диалога, состояние клиента |
| `knowledge.js` | Поиск ответов в БД: оборудование, FAQ, цены |

### 2. Сервисы — `src/core/services/`

| Файл | Назначение |
|------|-----------|
| `consultation.js` | Ответы на вопросы об оборудовании, технических характеристиках |
| `qualification.js` | Сбор данных: тип работ, площадь, высота, сроки |
| `pricing.js` | Расчёт ориентировочной стоимости, вывод прайса |
| `documents.js` | Приём файлов (чертёж, проект), сохранение в Supabase Storage |
| `contacts.js` | Контакты компании, перенаправление к менеджерам |

### 3. Адаптеры каналов — `src/channels/`

Каждый адаптер выполняет две задачи:
1. **Входящее:** формат канала → унифицированный объект `{ text, channel, language, clientId, files }`
2. **Исходящее:** текст ответа → формат канала (кнопки, HTML, голос)

### 4. Локализация — `src/i18n/`

Системные тексты бота (приветствия, кнопки, подтверждения) на 3 языках в JSON-файлах.

### 5. База данных — `src/db/`

Supabase (PostgreSQL) — 6 таблиц (см. ТЗ, раздел «Модель данных»).

## Поток обработки сообщения

```
1. Канал получает сообщение (текст / голос / кнопка)
       ↓
2. Адаптер преобразует в единый формат
       ↓
3. engine.js принимает сообщение
       ↓
4. language.js → определяет/проверяет язык
       ↓
5. dialog.js → определяет текущий шаг диалога
       ↓
6. intents.js → распознаёт намерение
       ↓
7. Вызов соответствующего сервиса
       ↓
8. Сервис обращается к БД (knowledge.js)
       ↓
9. Формирование ответа на языке клиента
       ↓
10. Адаптер преобразует ответ в формат канала
       ↓
11. Отправка клиенту + запись в dialog_logs
```

## API эндпоинты (веб-виджет)

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/health` | Проверка доступности |
| POST | `/api/chat/message` | Отправка сообщения |
| POST | `/api/chat/upload` | Загрузка файла (чертёж) |
| GET | `/api/chat/history/:clientId` | История диалога |
| WS | `/ws/chat` | WebSocket для реального времени |
