# 🤖 Bot Multispa

> Проект разработан по методологии Vibecoding

## 📁 Структура проекта

```
Bot Multispa/
├── docs/                          # Документация по этапам разработки
│   ├── 0_исследование.md          # Этап 0: Исследование рынка
│   ├── 1_идея.md                  # Этап 1: Описание идеи
│   ├── 2_техническое_задание.md   # Этап 2: Техническое задание
│   ├── 3_архитектура_проекта.md   # Этап 3: Архитектура
│   ├── 4_требования_к_дизайну.md  # Этап 4: Дизайн
│   ├── 5_запуск_продукта.md       # Этап 5: Локальный запуск
│   ├── 5.1_задачи.md              # Этап 5.1: Список задач
│   ├── 6_тестирование.md          # Этап 6: Тестирование
│   ├── 7_деплой.md                # Этап 7: Деплой
│   ├── 8_дебаггинг.md             # Этап 8: Дебаггинг
│   ├── 9_релиз.md                 # Этап 9: Релиз
│   └── 10_пост_релиз.md           # Этап 10: Пост-релиз
├── src/                           # Исходный код
│   ├── bot/                       # Логика бота
│   │   ├── handlers/              # Обработчики команд
│   │   ├── keyboards/             # Клавиатуры и кнопки
│   │   ├── services/              # Бизнес-логика
│   │   └── utils/                 # Утилиты
│   ├── db/                        # Работа с базой данных
│   │   ├── migrations/            # Миграции БД
│   │   └── models/                # Модели данных
│   └── config/                    # Конфигурация
├── tests/                         # Тесты
├── .env.example                   # Пример переменных окружения
├── .gitignore                     # Исключения Git
```

## 🔄 Цикл разработки

```
0. Исследование → 1. Идея → 2. ТЗ → 3. Архитектура → 4. Дизайн
→ 5. Локальный запуск → 5.1 Задачи → 6. Тестирование
→ 7. Деплой → 8. Дебаггинг → 9. Релиз → 10. Пост-релиз
```

## 🚀 Быстрый старт

1. Клонируй репозиторий
2. Скопируй `.env.example` в `.env` и заполни переменные
3. Установи зависимости: `npm install`
4. Запусти бота: `npm run dev`
